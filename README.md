## ğŸš€ Getting Started

### Prerequisites

- Node.js 18+ and npm
- PostgreSQL database (or Supabase account)
- Google Cloud account (Gmail API)
- Slack workspace and app
- Clerk account

### Installation

1. **Clone the repository**
```bash
git clone https://github.com/yourusername/inboxiq.git
cd inboxiq
```

2. **Install dependencies**
```bash
npm install
```

3. **Set up environment variables**

Create `.env.local` in the root directory:
```env
# Clerk Authentication
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_xxxxx
CLERK_SECRET_KEY=sk_test_xxxxx

# Gmail OAuth
GMAIL_CLIENT_ID=xxxxx.apps.googleusercontent.com
GMAIL_CLIENT_SECRET=GOCSPX-xxxxx

# Slack OAuth
SLACK_CLIENT_ID=xxxxx.slack.com
SLACK_CLIENT_SECRET=xxxxx
SLACK_SIGNING_SECRET=xxxxx

# Database
DATABASE_URL=postgresql://user:password@host:5432/database

# Gemini AI
GEMINI_API_KEY=xxxxx

# Redis (optional)
UPSTASH_REDIS_URL=xxxxx
UPSTASH_REDIS_TOKEN=xxxxx

# App URL
NEXT_PUBLIC_APP_URL=http://localhost:3000

# Encryption Key (generate with: openssl rand -base64 32)
ENCRYPTION_KEY=xxxxx
```

4. **Set up database**
```bash
npx prisma generate
npx prisma db push
```

5. **Run development server**
```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## ğŸ¨ Features in Detail

### 1. **Smart Email Categorization**
- Automatically identifies sender types (Professor, Recruiter, Manager, Peer, Newsletter)
- Classifies email purpose (Task, Submission, Meeting, Question, Information)
- Learns from user feedback to improve accuracy over time

### 2. **Multi-Factor Urgency Detection**
- **Keyword Analysis**: Detects "urgent", "ASAP", "deadline", "today"
- **Sender Priority**: Weights emails from professors/managers higher
- **Deadline Extraction**: Identifies and tracks due dates
- **Follow-up Detection**: Recognizes reminder emails (2nd, 3rd attempt)
- **Time Sensitivity**: Prioritizes emails requiring action within 24-48 hours

### 3. **Intelligent Notifications**
- **Immediate Alerts**: Urgent emails (score â‰¥6) sent to Slack instantly
- **Batched Digests**: Normal emails grouped every 30 minutes
- **Quiet Hours**: Respects user-defined do-not-disturb times
- **Channel Routing**: Can send to different Slack channels by category

### 4. **AI-Powered Draft Replies**
- Context-aware responses matching email tone
- Acknowledges deadlines and action items
- Professional formatting for work emails
- Casual tone for peer communications

### 5. **Customizable Rules**
- Define custom urgent keywords
- Whitelist/blacklist senders
- Adjust urgency thresholds
- Set digest frequency (15, 30, 60 minutes)
- Create custom categories

## ğŸ”’ Security

- **OAuth Tokens**: Encrypted using AES-256 before database storage
- **API Keys**: Stored in environment variables, never committed to git
- **Authentication**: Clerk provides enterprise-grade auth with MFA support
- **Rate Limiting**: Redis-based rate limiting on API endpoints
- **HTTPS Only**: All OAuth callbacks require HTTPS in production
- **CORS Protection**: Strict origin policies on API routes
